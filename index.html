Referencia rápida · Funciones DAX esenciales
SUM() Suma todos los valores de una columna numérica.
SUM(Ventas[Monto])
CALCULATE() Evalúa una expresión modificando el contexto de filtro.
CALCULATE(SUM(V[M]), Año=2024)
SUMX() Evalúa una expresión fila por fila y suma los resultados.
SUMX(Ventas, [Precio]*[Cant])
COUNTROWS() Cuenta el número de filas en una tabla o expresión de tabla.
COUNTROWS(Clientes)
DIVIDE() División segura que maneja el error de división entre cero.
DIVIDE([Ventas], [Objetivo], 0)
IF() Retorna un valor si la condición es verdadera, otro si es falsa.
IF([Meta]>0, "OK", "Alerta")
RELATED() Trae un valor de una tabla relacionada (lado "uno" de la relación).
RELATED(Producto[Categoría])
TOTALYTD() Acumula un valor desde el inicio del año hasta la fecha actual.
TOTALYTD([Ventas], Fechas[Fecha])
SAMEPERIODLASTYEAR() Devuelve el mismo período del año anterior para comparativas.
CALCULATE([V], SAMEPERIODLASTYEAR(F[F]))
ALL() Elimina todos los filtros de una tabla o columna.
CALCULATE([V], ALL(Dim_Región))
FILTER() Devuelve una tabla filtrada según una condición.
FILTER(Ventas, [Monto]>1000)
DISTINCTCOUNT() Cuenta los valores únicos en una columna.
DISTINCTCOUNT(Ventas[ClienteID])

Referencia

Glosario del Programa

Términos clave que encontrarás a lo largo de todos los módulos.

Business Intelligence (BI)
Conjunto de estrategias, procesos y tecnologías para transformar datos en información útil para la toma de decisiones empresariales.
Data-Driven
Enfoque en el que las decisiones se basan en el análisis de datos concretos en lugar de la intuición o la experiencia subjetiva.
ETL
Extract, Transform, Load. Proceso de extraer datos de fuentes, transformarlos y cargarlos en un destino analítico.
Power Query
Motor de transformación de datos en Power BI (y Excel) que usa el lenguaje M para limpiar, combinar y preparar datos.
Tabla de Hechos (Fact Table)
Tabla central del modelo que almacena métricas cuantitativas (ventas, cantidades) y claves foráneas hacia las dimensiones.
Tabla de Dimensión
Tabla que describe el contexto de los hechos: productos, clientes, fechas, geografía. Contiene atributos descriptivos.
Esquema Estrella
Diseño de modelo donde una tabla de hechos central se conecta directamente a múltiples tablas de dimensión. Óptimo para Power BI.
DAX
Data Analysis Expressions. Lenguaje de fórmulas de Power BI para crear medidas, columnas calculadas y tablas calculadas.
Medida (Measure)
Cálculo DAX que se evalúa dinámicamente según el contexto del reporte. No almacena datos, solo calcula al momento de visualizar.
Contexto de Filtro
El conjunto de filtros activos que determinan qué filas se evalúan en una medida DAX (segmentadores, filas de matriz, etc.).
Cardinalidad
Número de valores únicos en una columna. Alta cardinalidad = muchos valores únicos. Impacta el rendimiento del modelo.
KPI
Key Performance Indicator. Métrica clave que mide el desempeño de un objetivo empresarial específico.
Slicer / Segmentador
Control visual en Power BI que permite filtrar interactivamente los datos de un reporte mediante selección del usuario.
Power BI Service
Plataforma en la nube de Microsoft para publicar, compartir y colaborar en reportes e informes de Power BI.
Granularidad
El nivel de detalle de los datos en una tabla. Granularidad fina = un registro por transacción; gruesa = datos agregados.
Normalización
Proceso de organizar datos en múltiples tablas para reducir redundancia. Contrasta con la desnormalización usada en BI.
